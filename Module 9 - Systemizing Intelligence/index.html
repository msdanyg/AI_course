Claude Projects: Your Mission Archives

Every AI conversation starts with a blank slate. Claude doesn't remember your last conversation, your company, or your preferences. This creates the "repetition tax"—you spend the first 2-3 exchanges of every conversation rebuilding context.

Claude Projects solve this by creating persistent workspaces with three components:

📋 Custom Instructions

Persistent prompts that shape how Claude behaves in this context. Define your role, preferences, and output formats once.

Best for: Behavior rules

📁 Knowledge Files

Documents Claude can reference—PDFs, CSVs, text files. Upload battle cards, account profiles, documentation.

Best for: Reference materials

💬 Conversation History

Past chats that inform future responses. Claude builds understanding over time within the Project.

Best for: Ongoing context

🎯 Project Name

Clear, specific, action-oriented. "Q4 Competitive Intelligence" not "Work Stuff."

Best for: Organization

When to Create a Project

Create a Project When... Use Regular Conversation When...
You'll have 5+ conversations on the same topic It's a one-off question
You need consistent context across sessions You're doing quick research
You have reference documents to upload You're experimenting/exploring
The work requires specialized behavior Context doesn't need to persist

You are my [ROLE] for [DOMAIN].

BEHAVIOR:
- [How to approach tasks in this domain]
- [What to prioritize or avoid]

TERMINOLOGY:
- [Industry-specific language rules]
- [Terms to use or avoid]

OUTPUT PREFERENCES:
- [Format: bullets, tables, prose]
- [Length and structure]
- [Standard sections to include]
⚠️
Common Pitfall: Keep Custom Instructions under 500 words. Be specific, not exhaustive. Claude ignores overly long instruction sets.

Claude Skills: Your Mission Cards

While Projects provide persistent context, Skills provide portable methodology. They're reusable prompt templates that you can invoke with a simple trigger phrase—like pre-built mission cards for common operations.

Anatomy of a Skill

✓ Skill Structure 
SKILL NAME: Quick Competitive Analysis

TRIGGER: When I say "Quick comp on [COMPANY]"

INSTRUCTIONS:
1. POSITIONING: Their core value proposition (1 sentence)
2. STRENGTHS: Top 3 competitive advantages
3. WEAKNESSES: Top 3 vulnerabilities we can exploit
4. COUNTER: How we position against each weakness
5. VERIFY: List claims needing fact-checking

OUTPUT FORMAT:
Table I can paste into Slack

Example: Meeting Prep Skill

✓ Prep Brief Skill 
TRIGGER: When I say "Prep me for [MEETING/CLIENT]"

GENERATE:
1. Context refresh on the subject
2. Recent relevant history
3. Key priorities or objectives
4. Potential challenges or watch-outs
5. Recommended questions to ask

FORMAT:
- One page maximum
- Bullet points
- Include "Next Steps" section

Example: Brand Guidelines Skill

✓ Brand Application Skill 
TRIGGER: When I say "Apply brand to [CONTENT]"

REVIEW AND REVISE FOR:

Terminology:
- Use "users" or "accounts" (never "customers")
- Use "flexible hours schedules" (not "flex work")
- Use "Productivity Optimization" (not "Performance")

Tone:
- Lead with business outcomes, not features
- Emphasize "Insights, Not Oversight" philosophy

Formatting:
- No Oxford comma
- Avoid em-dashes unless necessary

OUTPUT: Revised content with [CHANGED] markers

This kind of Skill ensures brand consistency across all AI-generated content—emails, presentations, documentation. Build it once, apply it everywhere.

Skills vs. Projects: The Decision

Scenario Use Project Use Skill
Ongoing client work ✓
Quick formatting task ✓
Research with documents ✓
Repeatable analysis ✓
Team-shared methodology ✓
Deep strategic work ✓
💡
Best Practice: Combine them. Use Skills inside Projects for maximum efficiency. The Project provides context; the Skill provides methodology.

Gemini Gems: Specialized Recon Units

Gemini Gems are custom AI assistants with persistent instructions, optimized for Google Workspace integration. Think of them as specialized reconnaissance units—pre-configured for specific intelligence-gathering missions.

When Gems Excel

🔍 Real-time Research

Gems search with grounding—current, verified information Claude's training might miss.

Best for: Current data

📄 Google Workspace

Native integration with Docs, Sheets, and Slides. Export directly without copy-paste friction.

Best for: Workspace flows

👥 Collaborative Access

Share Gems with teammates. Everyone uses the same specialized researcher.

Best for: Team research

📰 Current Information

News, trends, recent data—anything where training cutoffs matter.

Best for: Fresh intel

Building a Gem

✓ Market Intel Gem 
NAME: Market Intel Researcher

INSTRUCTIONS:
You are a competitive intelligence researcher for ActivTrak.

YOUR FOCUS:
- Workforce analytics and employee monitoring software
- Key competitors: Teramind, Hubstaff, Insightful
- Industry trends in remote work, productivity, privacy

WHEN RESEARCHING:
- Always verify claims with multiple sources
- Include publication dates for statistics
- Flag anything older than 6 months as outdated
- Prioritize industry analyst reports

OUTPUT FORMAT:
- Lead with key findings (3-5 bullets)
- Detailed analysis follows
- End with "Confidence Assessment"
- Include source links for all claims
🔄
Decision Rule: Use Gems when you need current, grounded research with Google Workspace integration. Use Claude Projects for deep reasoning with documents.

Custom GPTs: Deployable Specialists

Custom GPTs are persistent assistants you build in ChatGPT that can be shared with others. They're like deployable specialists—AI teammates you can distribute to your organization.

The Key Differentiator: Distribution

While Projects, Skills, and Gems are primarily personal tools, Custom GPTs can be shared. Your entire sales team can use the same Battle Card Generator with the same knowledge base.

💡
Use Case: Build a "Battle Card Generator" GPT, upload competitive intelligence, and share it. Every sales rep gets consistent, accurate competitive positioning.

Building a Custom GPT

Component Purpose
Name Clear, professional identifier
Description What it does and who it's for
Instructions Detailed behavior guidance
Knowledge Uploaded reference files
Capabilities Enable/disable browsing, DALL-E, code
Conversation Starters Suggested first prompts

The Complete Decision Matrix

Need Primary Tool Why
Deep client work Claude Project Context + documents + history
Quick repeated tasks Claude Skill Portable methodology
Real-time research Gemini Gem Grounded, current data
Team-shared assistant Custom GPT Distributable, consistent
🏗️
Integration Example: Gemini Gem gathers current research → Claude Project synthesizes with historical analysis → Claude Skill structures the output → Custom GPT generates shareable battle card. Each tool does what it does best.

📋 Study Guide

🎯 Core Concept

Squadron Architect Model: Build persistent AI infrastructure that works for you even when you're not actively prompting. Move from giving instructions to building systems. Infrastructure scales. Individual prompts don't.

📖 Key Terms

Claude Project Persistent workspace with Custom Instructions, Knowledge Files, and Conversation History
Claude Skill Reusable prompt template invoked with a trigger phrase ("Quick comp on...")
Gemini Gem Custom AI assistant optimized for real-time research and Google Workspace
Custom GPT Shareable assistant in ChatGPT that can be distributed to teams
Repetition Tax Time wasted rebuilding context every conversation—eliminated by persistence tools

✈️ Decision Matrix

Need Primary Tool Why
Deep client work Claude Project Context + documents + history
Quick repeated tasks Claude Skill Portable methodology
Real-time research Gemini Gem Grounded, current data
Team-shared assistant Custom GPT Distributable, consistent

✓ Do's and Don'ts

✓ Do

✓ Create purpose-specific Projects (not mega-Projects)
✓ Keep Custom Instructions under 500 words
✓ Refresh knowledge files monthly
✓ Combine Skills inside Projects

✗ Don't

✗ Create Projects for one-off questions
✗ Write 2,000-word instruction sets
✗ Upload documents once and forget them
✗ Use wrong tool for task type

🔧 Maintenance Schedule

Frequency Actions
Weekly Review conversations for patterns, update Skills, check Gem accuracy
Monthly Refresh knowledge files, audit instructions, archive inactive Projects
Quarterly Evaluate expansion opportunities, share patterns with team
