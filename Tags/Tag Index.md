## Tag Usage Guidelines

### Do's
- Apply 3-5 relevant tags per note
- Use hierarchical tags (e.g., `#module/3-prompt-architecture` over just `#module`)
- Tag both the concept AND the tool when applicable
- Include learning level tags for assessment content

### Don'ts
- Avoid tag explosion (>10 tags per note)
- Don't create one-off tags without adding to this index
- Don't use spaces in tags (use hyphens)
- Don't mix Squadron metaphor with non-Squadron terms

### Tagging Examples

**Good:**
```markdown
---
tags:
  - module/3-prompt-architecture
  - prompt/xml-tags
  - squadron/mission-control
  - marzano/analysis
---
```

**Avoid:**
```markdown
---
tags:
  - prompt
  - claude
  - xml
  - important
  - todo
---
```
