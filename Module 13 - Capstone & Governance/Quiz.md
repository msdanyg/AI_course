### Question 1 (Multiple Choice)

**What does "AI drafts, humans send" mean in practice?**

A) AI should never create first drafts—humans should always write first
B) AI outputs require human review before external distribution, and humans own the consequences
C) AI can send emails automatically as long as a human wrote the prompt
D) Humans should always rewrite AI outputs completely before using them

**Correct Answer:** B

**Feedback for B (Correct):**
Exactly right. "AI drafts, humans send" is the core accountability principle. The AI is your collaborator, but you are the author. You review outputs before they leave your desk, and you own the consequences—"the AI wrote it" is never a valid excuse for errors.

**Feedback for other answers:**
- **Option A**: AI absolutely should create first drafts—that's one of its primary values. The principle is about review and accountability, not avoiding AI assistance.
- **Option C**: No. AI should never send anything automatically without human review. The human must actively review and approve before sending.
- **Option D**: Complete rewrites defeat the purpose. The principle requires review and editing where needed, not wholesale replacement.

### Question 2 (Scenario-Based)

**Scenario:** You've used Claude to analyze quarterly revenue data and generate a summary with key insights. The analysis will go to your VP in an important strategy meeting. What verification approach is most appropriate?

A) Quick skim—Claude is reliable for data analysis
B) Verify the methodology, spot-check 3-5 data points, confirm conclusions follow logically
C) Manually recalculate every number in the analysis
D) Ask Claude to verify its own work before sending

**Correct Answer:** B

**Feedback for B (Correct):**
This is the Spot-Check Protocol. For data analysis going to senior leadership, you should:
1. Verify the methodology is sound (did Claude approach it correctly?)
2. Spot-check 3-5 data points manually
3. Confirm conclusions follow logically from the data
4. Test edge cases if relevant

This balances thoroughness with efficiency—you're not recalculating everything, but you're catching potential errors.

**Feedback for other answers:**
- **Option A**: A quick skim is appropriate for low-stakes internal brainstorming, not analysis going to your VP.
- **Option C**: Manually recalculating everything defeats the purpose of using AI and isn't practical.
- **Option D**: AI cannot reliably verify its own work—human review is the governance mechanism.

### Question 3 (True/False with Explanation)

**TRUE or FALSE: The appropriate verification standard is the same for all AI outputs.**

**Correct Answer:** FALSE

**Feedback if FALSE (Correct):**
Correct. Verification standards scale with consequence:

- **Internal brainstorming:** Quick review for relevance
- **Draft internal documents:** Review for accuracy and tone
- **Customer-facing emails:** Full review + fact-check key claims
- **Data analysis conclusions:** Verify methodology + spot-check calculations
- **Public content:** Multi-person review + PR/Legal if needed
- **Financial/legal documents:** Expert review + compliance check

Applying the same scrutiny to everything wastes time on low-stakes work and may lead to under-reviewing high-stakes outputs.

**Feedback if TRUE (Incorrect):**
Verification standards should scale with consequence. A quick review is fine for internal brainstorming, but customer-facing content or financial documents require much more rigorous verification. Applying the same standard everywhere either wastes time or creates risk.

### Question 4 (Multiple Choice)

**What are the five phases of the Capstone Project (Squadron Mission)?**

A) Research → Draft → Edit → Review → Send
B) Capture → Research → Analysis → Draft → Verify
C) Plan → Execute → Monitor → Evaluate → Report
D) Brainstorm → Outline → Write → Proofread → Publish

**Correct Answer:** B

**Feedback for B (Correct):**
The five phases demonstrate end-to-end Squadron capability:

1. **Capture**: Voice memo or meeting notes → XML context (Flight Recorder integration)
2. **Research**: Recon & Radar → Fact Brief with citations (grounding)
3. **Analysis**: Code execution or logical analysis → documented methodology
4. **Draft**: Structured prompt → strategic brief (prompt architecture)
5. **Verify**: Human-in-the-Loop → accuracy, quality, governance, Red Team checks

Each phase maps to competencies from different modules.

**Feedback for other answers:**
- **Option A**: Missing the Capture phase (Flight Recorder integration) and Analysis phase (code execution). "Edit" isn't a distinct phase.
- **Option C**: These are project management phases, not the Squadron Mission workflow.
- **Option D**: This is a generic writing process that doesn't demonstrate Squadron capabilities.

### Question 5 (Scenario-Based)

**Scenario:** You've completed your capstone project draft. The Red Team check reveals that one of your supporting points is based on an assumption that might be wrong, and your competitor analysis is from 6 months ago. What should you do?

A) Note the limitations in the brief and proceed—the Red Team found theoretical concerns
B) Delete the uncertain supporting point and use only the remaining evidence
C) Verify the assumption with additional research, update the competitor data, then revise the brief
D) Ask Claude to generate alternative supporting points that don't require assumptions

**Correct Answer:** C

**Feedback for C (Correct):**
Red Team findings should trigger genuine improvements. If an assumption might be wrong, verify it. If data is stale, update it. The whole point of the Red Team check is to catch these issues before sharing. Address legitimate concerns, then finalize.

This demonstrates proper governance: Red Team → Identify issues → Resolve issues → Finalize

**Feedback for other answers:**
- **Option A**: "Note the limitations" is appropriate for genuine uncertainties, not for issues you could resolve. If you can verify the assumption or update the data, do it.
- **Option B**: Deleting evidence weakens your brief. Better to verify or replace with stronger evidence.
- **Option D**: The problem isn't the supporting points—it's that they need verification. Generating alternatives doesn't address the underlying issue.

### Question 6 (Multiple Choice)

**The "Transparency Prompt" asks the AI four questions before using output. Which is NOT one of them?**

A) What assumptions did you make?
B) What information would change your answer?
C) What similar outputs have you created before?
D) What should I verify independently?

**Correct Answer:** C

**Feedback for C (Correct):**
The Transparency Prompt asks:
1. What assumptions did you make?
2. What information would change your answer?
3. What are you least confident about?
4. What should I verify independently?

"What similar outputs have you created before?" is NOT part of this prompt. AI models don't have memory of previous sessions (in standard usage), and this question doesn't help you understand the current output's reliability.

**Feedback for other answers:**
- **Option A**: This IS part of the Transparency Prompt—understanding assumptions helps identify potential issues.
- **Option B**: This IS part of the prompt—knowing what would change the answer helps you assess robustness.
- **Option D**: This IS part of the prompt—knowing what to verify independently is essential for Human-in-the-Loop governance.
