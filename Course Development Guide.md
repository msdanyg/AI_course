# Course Development Guide

This document consolidates the instructional design framework and guidelines for the **"Solo Pilot to Squadron Leader"** GenAI training program.

---

## Course Overview

**Title:** From Solo Pilot to Squadron Leader: Practical AI for Knowledge Work

**Goal:** Transform ActivTrak employees from casual AI users into "AI Architects" capable of orchestrating complex reasoning workflows using Claude, Gemini, Granola and the Mac/Google ecosystems.

**Target Audience:** General Staff (Managers, ICs, Sales, CS, Ops)

**Format:** Hybrid (Conceptual Strategy + Hands-on Labs)

**Prerequisites:** Access to Claude Enterprise, Gemini for Workspace, and Granola

---

## The Transformation Story

Every piece of content must reinforce this journey:

### FROM: Solo Pilot
- One person, one AI tool, one-off prompts
- Reactive, improvisational, "I'll figure it out as I go"
- Generic outputs, wasted effort, inconsistent results
- Limited by their own expertise

### TO: Squadron Leader
- Commander of specialized AI agents with defined roles
- Proactive, systematic, reusable mission playbooks
- High-quality, consistent, verifiable outputs
- Amplified by orchestrated intelligence

## Squadron Metaphor Framework

**ALWAYS use these canonical mappings:**

| Concept | Squadron Metaphor | Notes |
|---------|------------------|-------|
| **Learner** | Squadron Leader | Primary identity; they architect and command |
| **Claude (reasoning)** | Mission Control / Command Center | The brain coordinating operations |
| **Gemini (research)** | Recon & Radar | Eyes and ears gathering intel |
| **Granola/Notes** | Flight Recorder & Comms Network | Captures signals from the field |
| **Skills/Templates** | Mission Cards / Squadron Playbook | Reusable operational procedures |
| **Projects/Workflows** | Missions / Mission Plans | Multi-step coordinated operations |
| **Data Safety (Green/Yellow/Red)** | Air Traffic Control / Tower Lights | Clearance system for takeoff |
| **Human responsibility** | "You clear missions for takeoff and own what leaves the runway" | AI drafts, humans send |

### Metaphor Conversion Rules
- Body parts (brain/eyes/memory) → Mission Control/Recon/Flight Recorder
- "Workflow as product" → "Mission as reusable operation"
- Mixed metaphors → Flatten to aviation or plain language
- **NEVER** introduce new metaphor families outside aviation/squadron

## Marzano's Taxonomy Application

Each module should address multiple cognitive levels:

### The Three Systems

1. **Self-System (Motivation)**
   - Why should learners care?
   - How does this connect to job success?
   - What's the emotional hook?

2. **Metacognitive System (Self-Regulation)**
   - How do learners monitor understanding?
   - What reflection points assess progress?
   - How do they know they've mastered it?

3. **Cognitive System (Processing)**
   - Level 1 - Retrieval: Recall terms and definitions
   - Level 2 - Comprehension: Explain in own words
   - Level 3 - Analysis: Compare, classify, identify errors
   - Level 4 - Knowledge Utilization: Apply to real scenarios

### Writing Learning Objectives

Format: `By the end of this module, learners will be able to [ACTION VERB] + [SPECIFIC CONTENT] + [CONTEXT/CONDITION]`

**Action Verbs by Level:**
- **Retrieval:** Define, list, recall, identify, name
- **Comprehension:** Explain, summarize, paraphrase, describe
- **Analysis:** Compare, contrast, classify, analyze, differentiate
- **Utilization:** Apply, construct, produce, demonstrate, create

## Content Development Workflow

### For Each Module, Generate:

1. **Module Brief** (Planning Doc)
   - Learning objectives (3-5 specific capabilities)
   - Key concepts to cover
   - Squadron metaphor mapping for this topic
   - ActivTrak use case examples
   - Prerequisites from previous modules
   - Setup for next module

2. **Core Content Package**
   - Written lesson (2,500-4,000 words)
   - Video script (1,500-2,000 words)
   - Flashcards (5-8 cards)
   - Study guide (1-page)

3. **Assessment Package**
   - Lab exercise (detailed walkthrough)
   - Quiz (5-7 questions with feedback)
   - Success criteria

4. **Instructor Resources**
   - Speaker notes
   - Common Q&A
   - Troubleshooting guide
   - Time estimates

## Format-Specific Guidelines

### Written Lessons
- **Length:** 2,500-4,000 words
- **Structure:** Hook → Core Concepts → Practical Application → ActivTrak Connection → Lab Preview
- **Tone:** Professional but conversational

### Video Scripts
- **Length:** 1,500-2,000 words (8-12 min)
- **Structure:** Opening hook (30 sec) → Core teaching (6-9 min) → Action preview (1-2 min)
- **Visual cues:** `[SCREEN: ...]`, `[PAUSE]`, `[DEMO: ...]`
- **Pacing:** One concept per 90 seconds max

### Flashcards
- **Front:** Question or key term
- **Back:** Concise answer (2-3 sentences max)
- **Focus:** Core concepts, not trivia

### Quizzes
- **Mix:** 60% multiple choice, 20% T/F with explanation, 20% scenario-based
- **Difficulty:** 70% recall, 30% application
- **Feedback:** Every answer explains why correct/incorrect

### Lab Exercises
- **Components:** Objective, Scenario, Step-by-step, Expected outputs, Troubleshooting, Stretch goals
- **Time:** 20-30 minutes

### Study Guides
- **Format:** Single-page visual summary
- **Include:** Key concepts, Squadron mapping, Do's and Don'ts, Quick reference, Next module preview

## ActivTrak Contextualization

### When to Use ActivTrak Examples

**ALWAYS for:**
- Product positioning and messaging
- Competitive intelligence workflows
- Customer success scenarios
- Internal data governance
- Brand voice and terminology

**SOMETIMES for:**
- General data analysis
- Writing and content creation
- Meeting intelligence

**RARELY for:**
- Foundational LLM mechanics
- Pure prompt engineering theory
- Tool setup and configuration

### Terminology Rules

**Always use:**
- "Users" or "accounts" (not "customers")
- "Flexible hours schedules" and "schedule adherence" (not "flex work")
- "Insights, Not Oversight" (privacy philosophy)
- "Productivity Optimization" (not "Performance Optimization")

**Formatting:**
- No Oxford comma
- No em-dashes unless absolutely necessary

## Quality Standards

### Every Piece of Content Must:

1. **Advance the Journey**
   - Connect to Solo Pilot → Squadron Leader transformation
   - Build on previous modules
   - Set up future capabilities

2. **Maintain Narrative Consistency**
   - Use canonical metaphor mappings
   - Avoid new metaphor families
   - Keep Squadron Leader as primary identity

3. **Be Pedagogically Sound**
   - Clear learning objective upfront
   - Theory → Practice → Application structure
   - Appropriate cognitive load
   - Spaced repetition of key concepts

4. **Include Practical Application**
   - Every concept has a "how-to" component
   - Hands-on practice opportunity
   - Real business scenario
   - Verifiable outcome

5. **Respect Data Governance**
   - Reference Traffic Light system (Green/Yellow/Red)
   - Reinforce "AI drafts, human sends" principle
   - Clarify PII usage rules

---

## What to Avoid

- ❌ Introducing new metaphor families outside aviation/squadron
- ❌ Mixing identities (stay with Squadron Leader)
- ❌ Generic AI examples that could be from any course
- ❌ Theory without practice
- ❌ Assuming prior knowledge
- ❌ Content overlap between modules
- ❌ Objectives without corresponding content/assessment
