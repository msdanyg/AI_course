The Calculator Problem

Here's a truth that surprises most people: Claude, the same system that can write sophisticated legal arguments and debug complex code, struggles with multiplication.

🔢 The Hallucination Test
✗ Without Code Execution
Question

What is 847 × 293?

AI Response
248,271
AI predicted digits that "look right"
✓ With Code Execution
Question

Use Python to calculate 847 × 293

AI Response
248,171
Python calculated the exact answer

Why This Happens

Remember from Module 1—Claude is a probabilistic reasoning engine. It doesn't retrieve answers or calculate them in a logic circuit. It predicts the most likely sequence of characters that should follow your input.

For simple math (2 + 2), the training data contains so many examples that the prediction is reliable. For complex or unusual calculations, the model is essentially guessing which digits "look right."

⚠️
The Danger: AI doesn't express uncertainty about math. It will confidently give you a wrong number. A plausible-sounding incorrect figure in a QBR can erode client trust and lead to bad decisions.

Code Execution: The Solution

The breakthrough insight is understanding two types of AI output:

Output Type How It Works Reliability
Probabilistic AI predicts likely text Variable—great for language, unreliable for math
Deterministic Code executes calculations Precise—computer actually computes the answer

The Analysis Tool

Claude's Analysis Tool (Code Execution) transforms data work. When enabled, Claude can:

  • Write Python code to process your data
  • Execute that code in a secure sandbox
  • Return calculated results that are mathematically precise
  • Generate visualizations based on actual data
Python 
# When you ask Claude to calculate with code:
import pandas as pd

# Load and analyze data
df = pd.read_csv('metrics.csv')
q2_total = df['Q2_Hours'].sum()
q3_total = df['Q3_Hours'].sum()

# Calculate exact percentage change
pct_change = ((q3_total - q2_total) / q2_total) * 100
print(f"Change: {pct_change:.2f}%")

Vibe Coding for Non-Technical Users

You might be thinking: "I'm not a programmer. I don't know Python."

Good news: you don't need to.

What Is Vibe Coding?

Vibe Coding is describing what you want in natural language—focusing on the outcome, the "vibe"—while AI handles all the technical implementation.

🎨 From Vibe to Visualization
Your Natural Language Request

"Create a bar chart comparing productivity by department. Make it look professional—dark background, sort from highest to lowest, add data labels. Use ActivTrak blue."

What Claude Does
  • Interprets "professional" → specific styling
  • Applies "dark background" → plt.style
  • Implements sorting logic
  • Uses #19316A for ActivTrak blue
  • Generates download-ready chart

The Refinement Loop

Vibe Coding works through iteration:

  1. Initial request: Describe the general outcome
  2. Review output: Check if it matches your vision
  3. Refine specifics: Add constraints as needed
  4. Verify accuracy: Ensure data is represented correctly
Refinement Prompts 
# Example refinement conversation:

"Good start. Now sort the bars from largest decline
to largest improvement."

"Color-code the bars: red for declines,
green for improvements."

"Add the exact percentage as a label on each bar."

"Make the font larger for projection."
🎯
You Direct; AI Implements. Each refinement, Claude adjusts the code and regenerates. You never need to know the technical syntax—just describe what you want differently.

The Verification Protocol

Here's the Squadron Leader protocol for trustworthy data analysis:

1

No Numbers Without Code

If Claude gives you a calculation, look for the code block. No code visible = number is a guess. Ask: "Show me the code you used."

2

Spot-Check Critical Figures

Pick 1-2 important numbers and verify them manually. Cross-reference with calculator or Excel. Takes 30 seconds, catches major errors.

3

Document Methodology

When sharing AI-generated analysis, note how it was produced. "Calculated using Python pandas" builds credibility and enables verification.

Common Pitfalls to Avoid

Pitfall Why It Happens Prevention
Confident wrong numbers AI doesn't express uncertainty Always verify code was used
Aggregation errors Sums/averages hide issues Check intermediate calculations
Missing data handling AI might ignore nulls Ask about data quality explicitly
Rounding confusion Display vs. calculation precision Specify precision requirements
📊
The ActivTrak Connection: When analyzing client metrics, remember "Insights, Not Oversight." Frame analysis supportively—ask about "patterns and trends" not "underperformers." The AI follows your framing.
