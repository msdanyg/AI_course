The Problem: Contextual Amnesia

Every standard chat with Claude or ChatGPT is like hiring a brilliant consultant who has amnesia. They wake up with no memory of your company, your role, your preferences, or your past conversations.

This creates three compounding problems:

1. Repetition Fatigue

You spend the first 2-3 exchanges of every conversation re-establishing basic context. "I work at ActivTrak. We sell workforce analytics. Our target market is mid-market enterprises..."

2. Consistency Drift

Without persistent instructions, outputs vary wildly based on how you happened to phrase things that particular day. Monday's competitive brief sounds nothing like Friday's because you forgot to mention tone preferences.

3. The Knowledge Ceiling

You can't build on previous work because every conversation starts at square one. That nuanced analysis you refined over three sessions last week? Gone. You're starting over.

The Solution: Persistent Mission Headquarters

Both Claude and ChatGPT offer solutions to contextual amnesia, but they work in fundamentally different ways.

Claude Projects: Hard Walls (Isolated Fortresses)

What it is: A Claude Project is a walled workspace where you can:

  • Upload up to 200 files (PDFs, docs, spreadsheets, code)
  • Set custom instructions that apply to every chat in that Project
  • Maintain separate chat histories that never leak into other Projects

The "Hard Wall" Analogy: Think of Claude Projects like classified military bases. What happens in Project A stays in Project A. The model has zero awareness of your other Projects. This isolation is a feature, not a bug—it prevents context pollution and maintains focus.

When Claude Projects Excel:

  • Long-term strategic work: Competitive intelligence tracking across quarters
  • Document-heavy tasks: Analyzing 50+ product specs, brand guidelines, or contract templates
  • Sensitive work: PII-heavy analysis where you need guaranteed isolation from other work
  • Complex custom instructions: Scenarios where you need 500+ words of behavioral guidance

ChatGPT Projects & Folders: Soft Boundaries (Flexible Command Centers)

What it is: ChatGPT offers two organizational layers:

  1. Folders (all tiers): Simple categorization for grouping related chats
  2. Projects (Plus/Team/Enterprise): Workspaces with file uploads and custom instructions

The "Soft Boundary" Analogy: ChatGPT Projects are more like open-plan offices with flexible dividers. The model has cross-conversation memory enabled by default, meaning insights from one chat can inform another (unless you explicitly disable memory).

When ChatGPT Projects Excel:

  • Iterative creative work: Content drafts that evolve over multiple sessions
  • Team collaboration (Enterprise): Shared Projects where teammates pick up where you left off
  • Memory-driven personalization: Scenarios where you want the AI to remember your writing style across all chats
  • Research aggregation: Pulling insights from multiple web searches into a unified knowledge base

Key Takeaways

  1. Projects eliminate the Context Rebuild Tax by providing persistent workspaces with pre-loaded files and instructions.
  2. Claude Projects = Hard Walls (isolated, high-capacity, doc-heavy). ChatGPT Projects = Soft Boundaries (flexible memory, team-friendly).
  3. The 80/20 Rule: Only create Projects for recurring work. One-time tasks belong in standard chat.
  4. Custom Instructions are Standing Orders—they apply to every chat in that Project. Start small (200 words) and iterate.
  5. Prune ruthlessly. Outdated files and over-instruction degrade AI performance. Monthly audits keep Projects sharp.
  6. Test immediately. Assume your instructions didn't work until proven otherwise. Run 3 test prompts after setup.

💡 Tips for Writing Effective Custom Instructions

Start with 4 key sections:

  1. ROLE: Who are you? What's your job title and company?
  2. CONTEXT: What does your company do? What's your positioning or philosophy?
  3. OUTPUT RULES: How should posts be structured? (length, format, tone)
  4. NEVER: What should Claude absolutely avoid? (forbidden language, bad patterns)

Best practices:

  • ✅ Be specific: "150-250 words, 3-5 short paragraphs" beats "keep it short"
  • ✅ Use examples: "Say 'reduced conflicts by 40%' not 'improved efficiency'"
  • ✅ List forbidden terms explicitly: "NEVER use 'synergy,' 'leverage,' 'disrupt'"
  • ✅ Start with 200 words—you can always add more later
  • ✅ Put critical rules at the END—Claude pays more attention to instructions at the bottom
  • ❌ Don't write 2,000 words covering every edge case (Claude ignores over-instruction)
ROLE: You are my LinkedIn content assistant. I'm a [Your Job Title] at ActivTrak, a workforce analytics company.

YOUR MISSION:
- Draft engaging LinkedIn posts that sound like me, not corporate marketing
- Highlight real business problems and how we solve them
- Use storytelling and examples, not just product features
- Keep posts authentic, conversational, and value-focused

POST STRUCTURE:
1. Hook (1-2 sentences that grab attention)
2. Body (problem, insight, or story)
3. CTA (comment prompt or link)

LENGTH & FORMAT:
- Target 150-250 words (3-5 short paragraphs)
- Use line breaks for readability (1-2 sentences per paragraph)
- No emojis unless I specifically request them
- End with a question or CTA to drive engagement

VOICE & TONE:
- Professional but approachable—like explaining to a colleague over coffee
- Avoid buzzwords ("synergy," "leverage," "disrupt")
- Use "we" when talking about ActivTrak, "I" when sharing my perspective
- Be specific with data: "reduced scheduling conflicts by 40%" beats "improved efficiency"

NEVER:
- Use surveillance language ("monitor employees," "track workers")
- Write generic LinkedIn fluff ("Excited to share," "Thrilled to announce")
- Include hashtags unless I ask for them
# LinkedIn Post Creation Instructions

## Core Directive
Generate authentic, data-driven LinkedIn content that establishes Daniel as a thought leader in workplace analytics, employee monitoring ethics, and HR technology. Posts should drive professional discussion while avoiding formulaic AI-generated patterns.

---

## Pre-Creation Requirements

### Input Specification
Before generating any post, confirm:
- **Topic/angle**: Specific workplace trend, research finding, or industry insight
- **Key data points**: Concrete statistics, percentages, or company examples to include
- **Tone preference**: Standard conversational, provocative/contrarian, or educational
- **Visual inclusion**: Will post include carousel, single infographic, or text-only?
- **Call-to-action preference**: Discussion question, follow request, or engagement prompt

If any element is unclear or missing, ask for clarification before proceeding.

---

## Writing Standards

### Voice & Tone
- Conversational, authentic, and human-first (contractions, informal transitions)
- Authoritative without being corporate or preachy
- Express genuine perspective while remaining professionally appropriate
- Vary sentence length dramatically (one-word declarations to multi-clause explanations)
- Use natural paragraph breaks to mirror spoken rhythm

### Structural Approach
- **Opening (lines 1-2)**: Bold, attention-grabbing statement or question that creates immediate value promise
- **Body (lines 3-10)**: Progressive disclosure of information in digestible chunks; concrete examples with specific company names and real statistics
- **Conclusion (final 2-3 lines)**: Clear synthesis or call-to-action; never end abruptly

### Formatting Elements
- Use arrow symbols (→, ➡️) for progression or connection
- Use numbered lists for step-by-step insights (5-7 items maximum)
- Strategic bolding for key statistics or insights (not overused)
- Line breaks between major ideas to create visual breathing room
- Emojis as visual anchors (1-2 maximum, placed strategically)

### Language Patterns
- **Active voice primary**: Use passive voice only where it sounds natural
- **Avoid jargon unless context-specific**: Replace formal/complex words with simpler alternatives
- **Specific over general**: Use actual company names, percentages, real-world examples
- **Action verbs**: Focus on verbs that create energy or urgency

---

## Content Themes to Leverage

### High-Performing Angles
- Failed or controversial workplace tech implementations (AT&T's "freeloader finder," etc.)
- Gaps between RTO mandates and actual compliance
- AI fairness and bias in HR systems
- Trust-building vs. surveillance in workplace monitoring
- Research-backed insights (Stanford WFH Research, SWAA dataset, etc.)
- Contrarian takes on workplace trends

### Data Sources to Reference
- Stanford Working from Home Research (wfhresearch.com)
- Survey of Working Arrangements and Attitudes (SWAA) dataset
- Forbes Technology Council, HR Reporter
- ActivTrak proprietary data where appropriate
- Industry reports with clear attribution

### Messaging Framework
- **Technology should enable trust, not replace it** (core principle)
- Measuring outcomes/impact is more effective than measuring visibility
- Transparency and consent matter in employee monitoring
- Context-dependent work requires flexible policy, not rigid surveillance

---

## Visual Content Standards

### Carousel Specifications (When Applicable)
- **Dimensions**: 1080 x 1350 pixels per slide
- **Font**: Century Gothic
- **Background**: White by default
- **Design elements**: Use ActivTrak brand colors (primary palette: #147bd1, #2ED4B5, #19316A, #14293F)
- **Typography**: Large, readable fonts (minimum 24px body text, 36px+ headers)
- **Layout**: Professional hierarchy with ample white space
- **Avoid**: ActivTrak logo, cluttered designs, unreadable small text

### Single Infographic Specifications
- **Dimensions**: 1080 x 1350 pixels
- **Design approach**: Tell a complete story through data visualization
- **Font**: Century Gothic
- **Readability first**: Optimize for mobile viewing at small sizes
- **Color palette**: Primary ActivTrak colors with complementary gradients
- **Avoid**: ActivTrak logo

### Quality Checklist
- All text readable at mobile size
- Color contrast meets accessibility standards
- Design avoids generic template appearance
- Data visualizations are clear and properly labeled
- Professional finish without unnecessary decoration

---

## Engagement Strategy

### Call-to-Action Approach
- End with thought-provoking question that invites genuine discussion
- Use phrases like "What's your take on...?" or "Am I off base here...?"
- Consider industry-specific CTAs that position Daniel as seeking expert input
- Avoid generic "Like and follow" language

### Engagement Triggers
- Share unique or contrarian perspectives backed by data
- Reference timely workplace trends or current events
- Provide actionable insight readers can apply immediately
- Create conversation starters that invite multiple viewpoints

---

## Quality Control Checklist

### Before Output
- [ ] No formulaic AI patterns (parallel sentence structures, robot-like transitions)
- [ ] Specific examples with real company names included
- [ ] Data points are accurate and properly attributed
- [ ] Contractions used naturally throughout
- [ ] Varied sentence length and structure
- [ ] Clear visual hierarchy with strategic line breaks
- [ ] Call-to-action is specific and invites engagement
- [ ] No Oxford commas used
- [ ] Em-dashes avoided unless absolutely necessary
- [ ] Terminology accurate (users/accounts, not customers; flexible hours schedules, not flex work)

### Visual Content Validation
- [ ] Dimensions correct (1080 x 1350)
- [ ] Font is Century Gothic
- [ ] Text readable at mobile size
- [ ] Background is white unless specifically designed otherwise
- [ ] No ActivTrak logo included
- [ ] Professional appearance without generic template feel

---

## Output Format

**Post deliverable**: Text only, placed in artifact with no additional explanation
**Visual deliverable**: HTML/code version optimized for copy-paste into LinkedIn, placed in artifact

Do not include commentary, process explanation, or meta-discussion of the post in the main response.

---

## Exception Handling

### When to Pause & Ask
- Topic lacks sufficient data or research to support claims
- Required context missing (audience segment, campaign timing, etc.)
- Request conflicts with brand positioning or accuracy standards
- Visual specifications unclear or require clarification

### Confidence Thresholds
- Only include claims you're 90%+ confident in
- Flag uncertain information with [TBD] or request clarification
- Use recent/current information (search if topic may have updated since January 2025)

What it is: The 5-10 minutes wasted at the start of every chat re-explaining your role, company, preferences, and context

Cost: Hours per month of repeated explanations

Solution: Claude Projects and ChatGPT Projects/Folders maintain persistent context

Platform Comparison

Feature Claude Projects ChatGPT Projects
Memory Model Hard Walls (100% isolated) Soft Boundaries (cross-conversation memory)

Best For PII work, sensitive data, compartmentalized tasks Creative work, team Projects, iterative drafts

Decision Tree: When to Create a Project

  1. Do you need persistent files or custom instructions? → No: Use standard chat
  2. Is this work recurring (weekly/monthly)? → One-time: Standard chat is fine
  3. Does it involve sensitive PII? → Yes: Use Claude Projects
  4. Want AI to learn your style across all chats? → Yes: Use ChatGPT Projects

The 5-Minute Test: "Would I spend 5+ minutes re-briefing the AI on this context?" If no, skip the Project.

Building a Claude Project (Quick Steps)

  1. Create: Click "Projects" → "New Project" → Name it
  2. Upload: Add 3-5 core documents
  3. Instruct: Write 200-word custom instructions (Role, Context, Output Rules, Never Do)
  4. Test: Run 3 test prompts (role memory, file usage, terminology compliance)
  5. Refine: Adjust instructions based on what Claude forgets

Common Mistakes to Avoid

📁
The Junk Drawer
Problem: Dumping 200 random files into one Project.
Fix: Create focused Projects with 5-15 signal-rich files max.
📝
Over-Instructing
Problem: Writing 2,000-word custom instructions trying to anticipate everything.
Fix: Start with 200 words; add instructions sparingly based on actual problems.
🗑️
Never Pruning
Problem: Outdated files accumulate, diluting retrieval quality with stale information.
Fix: Monthly audit—archive or remove files older than 90 days.
🧪
Forgetting to Test
Problem: Assuming custom instructions work without verification.
Fix: Run 3 test prompts immediately after setup to validate behavior.
🎯
One-Time Projects
Problem: Creating Projects for one-off tasks like vacation planning or homework.
Fix: Projects are for recurring, systematic work only. Use standard chat for one-offs.

Understanding RAG in Claude Projects

What It Is: Retrieval-Augmented Generation—Claude's "research assistant" that searches your files semantically

Mode Threshold How It Works
Full Context <200K tokens (~150 pages) Claude reads ALL files into memory
RAG Mode >200K tokens Claude searches indexed documents for relevant chunks

Why It Matters: Enables 10x capacity expansion (200 files instead of 20)

Optimization Tips: Use Markdown format, structure documents with clear headers, remove redundant content, keep file names descriptive, don't over-upload

ChatGPT Memory Settings

  • Cross-Conversation Memory: ChatGPT remembers facts/preferences across ALL chats. Best for personal creative work.
  • Project-Only Memory: Information stays siloed within that Project. Best for sensitive client work.
  • Decision Rule: Sensitive data/clients → Project-only memory. Personal creative work → Cross-conversation memory.
  • Important: Shared Projects (Business/Enterprise) automatically use project-only memory to prevent team data leakage

Key Terminology

  • Context Rebuild Tax: Time wasted re-explaining context every chat (5-10 min per session)
  • Hard Walls (Claude): 100% isolation between Projects—what happens in Project A stays in Project A
  • Soft Boundaries (ChatGPT): Cross-conversation memory bleeds between chats (unless project-only mode)
  • Custom Instructions: Persistent "Standing Orders" for every chat in a Project (start with 200 words)
  • RAG (Retrieval-Augmented Generation): Technology that lets Claude search large file sets semantically; auto-activates at 200K tokens
  • Full Context Mode: Claude reads all files into memory (<200K tokens)
  • Vectorization/Embeddings: Converting documents into "numerical fingerprints" for semantic search
  • Cross-Conversation Memory: ChatGPT remembers facts across all chats (global)
  • Project-Only Memory: Information stays siloed within one Project (for sensitive data)
  • Squadron Metaphor: Projects = Mission Headquarters with pre-briefed AI; RAG = Intelligence Archive
