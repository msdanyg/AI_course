**NARRATOR:**

[SCREEN: Simple diagram showing "Solo Pilot" working with "Amnesiac AI"]

This creates three compounding problems.

[SCREEN: Text overlay for each problem as discussed]

**Second: Consistency Drift.** Without persistent instructions, outputs vary wildly. Monday's competitive brief sounds nothing like Friday's because you phrased your preferences differently.

**Third: The Knowledge Ceiling.** You can't build on previous work. That analysis you refined over three sessions last week? Gone. You're starting over from scratch.

[SCREEN: Animation showing work being erased, Groundhog Day style]

In the Squadron metaphor, this is like your co-pilot forgetting how to fly the aircraft every single morning. Imagine briefing them on radio frequencies, mission objectives, and flight controls before every single takeoff.

Here's the critical feature: **Projects are 100% isolated.** What happens in Project A stays in Project A. Claude has zero awareness of your other Projects. This prevents context pollution and maintains laser focus.

[DEMO: Quick tour of Claude interface showing Projects sidebar, file upload area, custom instructions]

Claude Projects excel at long-term strategic work, document-heavy analysis, and anything involving sensitive data where you need guaranteed isolation.

[SCREEN: Switch to ChatGPT Projects side]

**ChatGPT Projects have soft boundaries with flexible dividers.** You can upload files and set custom instructions, but ChatGPT has cross-conversation memory enabled by default. Insights from one chat can inform another, even outside your Projects.

[DEMO: Quick tour of ChatGPT interface showing Projects, folders, memory settings]

ChatGPT Projects excel at iterative creative work, team collaboration on shared Projects, and scenarios where you want the AI to learn your style across all conversations.

## SECTION 3: WHEN TO USE WHAT (90 seconds)

**NARRATOR:**

[SCREEN: Decision tree flowchart]

Here's the decision framework.

**Start here:** Do you need to upload files or set persistent custom instructions?

If no—use standard chat. Projects are overkill for one-time tasks.

If yes—continue.

**Next question:** Is this work ongoing—weekly or monthly—or one-time?

If one-time, standard chat is fine. Upload files inline.

If ongoing, create a Project.

**Next question:** Does this involve sensitive personal data or information that must stay isolated?

[SCREEN: Examples: PII, contract analysis, internal financial data]

If yes, use **Claude Projects**. The hard walls guarantee no leakage.

If no, continue.

**Final question:** Do you want the AI to learn your style and remember preferences across all your chats?

[SCREEN: Examples: Creative writing, personal productivity, content creation]

If yes, use **ChatGPT Projects** with cross-conversation memory enabled.

If no, use **Claude Projects** for focused, single-purpose work.

## SECTION 4: BUILDING YOUR FIRST CLAUDE PROJECT (2 minutes)

**NARRATOR:**

[SCREEN: Screen recording of Claude interface]

Let's build a practical Claude Project for ActivTrak competitive analysis—a task most roles do monthly.

[DEMO: Click "Projects" in sidebar → "New Project"]

**Step 1: Create the Project.** I'm naming this "Competitive Intel HQ."

[DEMO: Show naming screen]

**Step 2: Upload your knowledge base.** Ask yourself: What documents do I reference every time I do competitive analysis?

[DEMO: Click "Add Content" → "Upload Files"]

Good candidates: competitor feature matrices, past competitive briefs you've written, ActivTrak positioning guides, analyst reports.

Bad candidates: 200-page annual reports with too much noise, or files you only need once per year.

[DEMO: Select and upload 5-6 sample files, wait for processing indicator]

Start small—5 to 10 core documents. You can always add more later.

[PAUSE]

**Step 3: Write your Standing Orders.** This is the custom instructions box. These rules apply to every chat in this Project.

[DEMO: Click into custom instructions field]

Here's the template I use:

[SCREEN: Show custom instructions text as it's typed]

```
ROLE: You are a Product Marketing Manager at ActivTrak, a workforce analytics company.

CONTEXT: ActivTrak competes with Teramind, Hubstaff, and Insightful in the mid-market enterprise space (50-500 employees).

YOUR MISSION:
- Analyze competitor moves (pricing changes, feature launches, positioning shifts)
- Identify gaps we can exploit
- Draft positioning that highlights our privacy-first philosophy ("Insights, Not Oversight")

OUTPUT RULES:
- Start with 3-bullet executive summary
- Use tables for feature comparisons
- Avoid fear-based messaging (never "catch bad employees")
- Length: 500-750 words unless otherwise specified

NEVER:
- Recommend surveillance-style positioning
- Use terms like "tracking" without context
- Ignore our Google Workspace and Microsoft 365 integration advantage
```

[DEMO: Save custom instructions]

**Pro tip:** Start with 200 words. If Claude consistently forgets something, add it. If it follows something without being told, remove it. Prune ruthlessly.

**Step 4: Test your Project.** Let's verify the instructions took.

[DEMO: Start a new chat in the Project]

I'll type: "Teramind just announced a new feature: automated screenshot analysis with AI. How should ActivTrak respond?"

[DEMO: Submit and show Claude's response]

Notice: Claude immediately remembered ActivTrak's positioning, contrasted with our privacy philosophy, formatted as a 3-bullet executive summary, and avoided fear-based language. All without me re-explaining any of that.

[SCREEN: Checkmarks appearing next to each requirement]

That's the power of persistent context. No rebuild tax.

## SECTION 5: BUILDING YOUR FIRST CHATGPT PROJECT (90 seconds)

**NARRATOR:**

[SCREEN: Switch to ChatGPT interface recording]

Now let's build a ChatGPT Project for Customer Success email drafting—a task CS teams do daily.

[DEMO: Click "Projects" → "New Project"]

I'm naming this "CS Email Drafting HQ."

[DEMO: Drag files into upload area]

For reference files, I'm uploading: top-performing CS emails for tone examples, customer persona docs, product feature one-pagers, and objection handling scripts.

[DEMO: Click into custom instructions]

Here's my template:

[SCREEN: Show instructions as typed]

```
I am a Customer Success Manager at ActivTrak.

AUDIENCE: Mid-market enterprises using ActivTrak for productivity optimization.

TONE:
- Warm but professional
- Empathetic to customer pain points
- Solution-focused

TERMINOLOGY:
- Use "users" or "accounts" (NOT "customers" for end users)
- Use "flexible hours schedules" (NOT "flex work")
- Use "insights" (NOT "tracking" unless in technical context)

OUTPUT FORMAT:
- Subject line (50 chars max)
- Email body (150-200 words)
- Call-to-action (1 sentence)

NEVER use surveillance language or mention competitors negatively.
```

[DEMO: Configure memory settings]

For personal Projects, keep cross-conversation memory ON so ChatGPT learns your writing style over time.

For shared Projects with your team, set to "Project-only memory" to prevent your teammates' chats from affecting yours.

[DEMO: Test with a prompt: "Draft an email helping a customer understand schedule adherence reports."]

Notice how ChatGPT applied the tone, terminology, and format rules without me repeating them.

## SECTION 7: COMMON MISTAKES (60 seconds)

**NARRATOR:**

[SCREEN: Warning icon with "5 Common Mistakes"]

Let's cover the five mistakes I see most often.

**Mistake 1: The Junk Drawer.** Creating one giant Project called "Work Stuff" with 200 files dumped in. The AI can't distinguish signal from noise. **Fix:** Create focused Projects with 5 to 15 files max.

**Mistake 2: Over-Instructing.** Writing 2,000 words of custom instructions covering every edge case. Claude and ChatGPT ignore instructions that drown in noise. **Fix:** Start with 200 words. Add only when the AI consistently makes the same mistake.

**Mistake 3: Never Pruning.** Adding files but never removing outdated ones. Your Q1 2025 competitive brief is still in your Q4 2025 Project, confusing the AI. **Fix:** Monthly audit. Archive files older than 90 days unless evergreen.

**Mistake 4: Forgetting to Test.** Setting up a Project and assuming it works. Instructions are hard to get right on the first try. **Fix:** Run 3 test prompts immediately after setup.

**Mistake 5: Using Projects for One-Time Tasks.** Creating a Project for something you'll do once, like "Plan my vacation." **Fix:** Projects are for recurring work. If you won't revisit it in 30 days, skip the Project.
